# Audio Carousel

[English](README.md) | [日本語](README.ja.md) | [简体中文](README.zh-Hans.md)

設定したデバイスリストをグローバルホットキー1つで巡回し、システムの既定の音声出力デバイスを切り替える、軽量な Windows タスクトレイユーティリティです。

[PeekDesktop](https://github.com/shanselman/PeekDesktop) からインスパイアされた設計：インストーラ不要・単一実行ファイル・設定はポータブル。

---

## 特長

- **ホットキー1つで切替。** ゲーム中・会議中・ブラウザ閲覧中など、どのアプリを使っていてもホットキーを押すだけで、システムの既定音声出力が次のデバイスへ切り替わります。サウンド設定パネルを開く必要はありません。
- **巡回するデバイスを自分で選べる。** 一般的な Windows PC には HDMI（モニタごと）・ペアリング済み Bluetooth ヘッドセット・仮想オーディオケーブルなど多数の出力先が並んでいます。Audio Carousel は **あなたが選んだデバイスだけ** を巡回するので、巡回ループは短く予測可能です。
- **切り替えた瞬間が見える。** アクティブモニタの右下にトーストが表示され、新しいデバイス名がわかります。トレイアイコンのツールチップにも現在のデバイス名が出ます。
- **トレイ常駐・マウスでも操作可。** トレイアイコンを左クリックすると巡回（ホットキーと同じ）。右クリックでメニュー：現在のデバイス、次へ巡回、設定、バージョン情報、終了。
- **柔軟なホットキー。** 修飾キー組合せ（`Ctrl+Alt+A`）はもちろん、プログラマブルキーボードやマクロパッドで人気の `F13`〜`F24` も使えます。
- **10言語 UI。** English / 日本語 / 简体中文 / 繁體中文 / Español / Français / Deutsch / Português (Brasil) / Русский / 한국어。Windows の表示言語から自動選択。設定からいつでも手動切替できます。
- **ポータブル。** 設定は実行ファイルと同じフォルダの `audio-carousel.json` に保存されます。フォルダごと USB メモリや OneDrive、別の PC に持っていけば、ホットキー・デバイスリスト・言語設定もそのまま引き継げます。
- **Windows 起動時に自動起動（任意）。** 「Windows 起動時に開始」を有効にすると、現在のユーザー専用の `HKCU\...\Run` キーにエントリが追加されるだけ。管理者権限・サービス・タスクスケジューラは一切使いません。
- **インストーラ不要・管理者権限不要・.NET ランタイム不要。** 実行ファイルを好きなフォルダに置いて起動するだけ。自己完結型ビルドなので追加インストールは何も必要ありません。
- **オフラインデバイスにも対応。** ケーブルを抜いた・電源を切った・現在接続されていないデバイスは設定画面でグレーアウト表示され、巡回時には自動でスキップされます。Bluetooth ヘッドホンに特に便利で、巡回リストに常時登録しておけば、実際に接続している時だけローテーションに自動で復帰します。

## ダウンロードと起動

1. [Releases](https://github.com/kai-rin/audio-carousel/releases) ページから `AudioCarousel.exe` をダウンロードし、好きなフォルダに配置します（例：`C:\Tools\AudioCarousel\`）。
2. ダブルクリックで起動します。

   > **初回起動時の注意：** バイナリは署名されていないため、Windows SmartScreen が「PC が保護されました」と表示することがあります。**詳細情報 → 実行** をクリックしてください。あるいは起動前に .exe を右クリック → **プロパティ** → **ブロックの解除** にチェック → **OK** でも構いません。理由は下の *免責事項* を参照。

3. タスクトレイ（時計の近く）にアイコンが表示され、初回起動時は自動的に **設定** ウィンドウが開きます。

## 初回設定

設定ウィンドウで：

1. **巡回したい音声出力デバイスを追加。** *デバイスを追加* をクリックしてドロップダウンから選びます。よくある組合せ：スピーカー・ヘッドセット・HDMI TV・Bluetooth イヤホン。2 つ以上あればホットキーが活きます。
2. **必要なら並べ替え。** *上へ移動* / *下へ移動* で巡回順を決めます。
3. **ホットキーを設定。** ホットキー欄をクリックして、使いたいキーを押すだけ（例：`F16`、`Ctrl+Alt+A`、`Win+Shift+S`、未使用のもの）。*クリア* で解除できます。
4. **（任意）UI 言語。** 既定は *自動*（Windows 設定に追従）。明示的に選ぶこともできます。
5. **（任意）Windows 起動時に開始。** チェックを入れると、サインイン時に Audio Carousel が自動的に立ち上がります。
6. **OK** をクリック。

## 日常的な使い方

- **ホットキーを押す** → 巡回リストの次のデバイスへジャンプ。アクティブモニタの右下にトーストが出ます。
- **トレイアイコンを左クリック** → ホットキーと同じ動作。
- **トレイアイコンを右クリック** → メニューが開きます：
  - 現在のデバイス名（情報表示・クリック不可）
  - **次へ巡回** — ホットキーと同じ
  - **設定** — 設定ウィンドウを再度開く
  - **Windows 起動時に開始** — 設定を開かずに切替
  - **バージョン情報**
  - **終了** — アプリを終了

## 設定ファイルとデータの保存先

- **設定ファイル：** 実行ファイルと同じフォルダの `audio-carousel.json`。手動で編集してもよいですし、設定画面から変更しても構いません。
- **レジストリ：** 書き込みは「Windows 起動時に開始」を ON にしたときの `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` のみ。OFF にすると削除されます。影響を受けるのは現在のユーザーだけで、管理者権限は不要。
- **常駐サービス・スケジュールタスク・テレメトリは一切なし。** アプリの実体は exe ファイル 1 つだけです。

## アンインストール

1. トレイアイコンを右クリック → **設定** → *Windows 起動時に開始* のチェックを外す → **OK**（レジストリエントリが削除されます）。
2. トレイアイコンを右クリック → **終了**。
3. `AudioCarousel.exe` と `audio-carousel.json` の入ったフォルダを削除。

手順 1 を飛ばしてしまった場合は、`regedit` で `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` の `AudioCarousel` 値を削除すれば後片付けできます。

## 動作環境

- Windows 10 1809 以降、または Windows 11
- x64
- 管理者権限不要
- .NET ランタイム不要（自己完結型ビルド）

## 免責事項

Audio Carousel は既定の音声エンドポイントを切り替えるために、非公開の Windows COM インターフェース である `IPolicyConfig` を呼び出します。これは SoundSwitch / EarTrumpet / NirCmd など多くの類似ツールが採用している手法で、Windows Vista から Windows 11 まで安定して動作してきました。ただし非公式 API のため、Microsoft が将来の Windows アップデートで予告なく変更・削除する可能性はあります。将来の Windows アップデートで Audio Carousel が動かなくなった場合は、Issue を立ててください。

リリースバイナリにはコード署名を行っていません（コード署名証明書には継続的な費用がかかるため、無償配布の本プロジェクトでは現状導入を見送っています）。バイナリのレピュテーションが蓄積されるまでは、初回起動時に Windows SmartScreen の警告が出ます。*ダウンロードと起動* の *初回起動時の注意* を参照してください。

# 開発者向け

ここから下はソースからビルドする人向けの情報です。エンドユーザーは上のリリースバイナリをダウンロードするだけで、以下の作業は一切必要ありません。

## 技術スタック

- C# / .NET 9 / WinForms（`net9.0-windows`）
- テスト: xUnit
- `NAudio.Wasapi`（`NAudio.CoreAudioApi` 名前空間）＋ `src/AudioCarousel/Audio/` 配下のインライン `IPolicyConfig` COM 宣言

## ビルドとテスト

Windows 上で .NET 9 SDK が必要です。

```bash
dotnet build
dotnet test
dotnet format          # .editorconfig に従ってホワイトスペース・改行コードを整形
```

## 単一実行ファイルの発行

```powershell
pwsh ./scripts/publish.ps1
```

出力：`publish/AudioCarousel.exe`（約 108 MB、自己完結型）。

リリース用 exe を生成する**唯一の正規手順**は `scripts/publish.ps1` です。フラグは慎重に調整されているので、`dotnet publish` の引数を手で組み立てないでください。

> **バイナリサイズについて。** Windows Forms は NativeAOT と非互換（`NETSDK1175`）であり、トリミングは `NAudio.CoreAudioApi` が依存する COM 相互運用機構を取り除いてしまいます。そのため本スクリプトは **トリミング無効・自己完結型・JIT・単一ファイル発行** を採用しており、約 108 MB の実行ファイルが生成されます。このサイズは「.exe をどこに置いても、.NET ランタイム不要で動く」という利便性の対価で、意図的に選択したトレードオフです。
